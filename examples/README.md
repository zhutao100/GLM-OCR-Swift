# Examples

This folder holds the checked-in example corpus plus the artifact families used by the parity workflow.

## Artifact families

| Path | Purpose | Update policy |
|---|---|---|
| `source/` | Input corpus mirrored from the upstream GLM-OCR examples. | Update only when the source corpus intentionally changes. |
| `reference_result/<name>/` | OCR output from the upstream reference implementation. | Update only when the upstream parity contract changes. |
| `reference_result_notes/<name>/` | Human notes about upstream quirks or adjudication details. | Update when the note itself changes, not to hide regressions. |
| `golden_result/<name>/` | Human-verified target used for scoring/calibration. | Update only with deliberate human-reviewed adjudication. |
| `result/<name>/` | Current repo output generated by `scripts/run_examples.sh`. | Update only after `scripts/verify_example_eval.sh` shows no significant unexplained regression. |
| `eval_records/latest/` | Latest scored verification snapshot for the checked-in `result/` set. | Refresh in the same commit as accepted `result/` changes, or in an explicit parity verification commit with zero-delta results. |

## Output contract matrix

| Behavior | Status | Notes |
|---|---|---|
| Markdown region order | Guaranteed | Follows the layout reading order produced by the runtime formatter. |
| Markdown image placeholder replacement | Repo-owned | `MarkdownImageCropper` materializes cropped image files only when an export directory exists. |
| Examples-style block-list JSON key order | Guaranteed | Stable serialization is protected by `OCRBlockListExportStableJSONTests`. |
| Block labels for `table`, `formula`, and `image` | Guaranteed | Export preserves the native semantic label instead of collapsing everything to `text`. |
| `OCRDocument` polygon geometry | Best-effort | Preserved when layout geometry is available; formula OCR may still use bbox crops even when polygons are exported. |

## Rebaseline rules

- Run `scripts/verify_example_eval.sh` before accepting `examples/result/*` changes.
- Refresh `examples/eval_records/latest/*` in the same commit as accepted `examples/result/*` updates so the evidence stays attached.
- Treat `scripts/lib/_parity_defaults.sh` as the checked-in parity contract for model revisions and the parity preset.
- Use `GLMOCR_RUN_EXAMPLES=1 swift test --filter LayoutExamplesParityIntegrationTests` to exercise the current protected subset (`GLM-4.5V_Page_1` PDF and `table` PNG).
